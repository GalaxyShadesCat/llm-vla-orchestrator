# 1. Project Overview

We propose to develop a fully autonomous robotic chess-playing system powered by a hierarchical Vision-Language-Action (VLA) architecture. The system integrates multimodal perception, high-level reasoning, and robotic manipulation to enable a robot to perceive a chessboard, determine the next move, and physically execute it without human intervention.

The key idea is to separate high-level strategic reasoning from low-level motor control. A chess engine evaluates the current board state and proposes candidate moves, while a Vision-Language Model (VLM) translates the selected move into a natural language instruction that can be interpreted by the robot. A Vision-Language-Action (VLA) model then performs the physical manipulation required to execute the instruction on the chessboard.

The system operates in a closed feedback loop, with actions verified after execution. If a mistake occurs, a validator agent detects the failure and requests a retry, creating a self-correcting robotic system.

Although our implementation focuses on chess, the broader goal is to demonstrate a reusable architecture for integrating VLM reasoning with VLA execution in real-world robotics tasks.

# 2. System Architecture and Agent Flow

The system follows a perception → reasoning → execution → validation loop.

### 1. Vision Module (Board State Extraction)

A camera captures RGB images of the chessboard. A vision module processes the image to detect the location and type of each piece, then converts this into a symbolic board representation (for example, a coordinate grid or FEN format).

This structured board state serves as input to the reasoning agent and enables deterministic move verification.

---

### 2. Chess Reasoning Agent (VLM + Chess Engine)

The reasoning module determines the next move to play based on the current board state.

The symbolic board representation extracted from the vision module is provided to the open-source chess engine Stockfish, which generates candidate moves along with evaluation scores.

Instead of always selecting the single optimal move, the system samples from the top-ranked candidate moves. This allows the reasoning agent to vary move quality and maintain a competitive but balanced game rather than producing deterministic optimal play.

A Vision-Language Model converts the selected move into a natural language command suitable for robotic execution. For example:

"Move the white bishop from c1 to g5."

This step transforms the structured move output from the chess engine into an instruction that can be grounded and executed by the VLA model.

---

### 3. VLA Execution Model (π₀)

The VLA model receives the natural language instruction along with visual observations of the board.

It grounds the instruction in the scene and performs the required manipulation by:

* locating the specified piece
* grasping it with the robotic arm
* placing it at the target square

To improve performance, we will fine-tune the π₀ model on approximately 100 real-world demonstrations of chess piece manipulation, captured using robots such as Cobot Magic or WidowX.

---

### 4. Validator Agent (VLM)

After execution, a new image of the board is captured and processed by the vision module to reconstruct the updated board state.

A Vision-Language Model (VLM), implemented using a large language model such as ChatGPT, acts as the validator agent.

The validator receives:

* the original board state
* the intended move instruction generated by the reasoning agent
* the observed board state after execution

Using this information, the VLM evaluates whether the robot correctly executed the intended move. The model reasons about whether the correct piece moved, whether it was placed on the intended square, and whether any unintended pieces were displaced.

Using a VLM for this step allows the system to interpret discrepancies between the intended command and the observed board state in a flexible way, enabling the system to detect and reason about partial or ambiguous execution errors.

If the validator determines that the move was executed correctly, the system proceeds to the next turn.

If the move was executed incorrectly (for example due to grasp failure, incorrect placement, or misidentification), the validator instructs the reasoning agent to regenerate the command and retry the action.

This creates a self-correcting feedback loop that allows the system to recover from manipulation errors during gameplay.

# 3. Methodology

## Data Collection and π₀ Fine-Tuning

We will collect approximately 100 demonstrations of chess-piece pick-and-place actions using a robotic arm.

Each demonstration will include:

* camera observations
* natural language command
* executed robot trajectory

These demonstrations will be used to fine-tune the π₀ model for reliable manipulation of chess pieces.

---

## Vision Pipeline Development

A perception module will be implemented to convert camera images into symbolic board states.

Possible approaches include:

* colour-based segmentation
* object detection models for chess pieces
* coordinate mapping between the camera frame and the board grid

This module enables both move planning and automated validation.

---

## Prompt Engineering for VLM Agents

Prompt templates will be developed for the reasoning and validator agents to ensure:

* clear instruction formatting
* reliable move interpretation
* consistent system behaviour

---

## System Integration

The following components will be integrated into a unified pipeline:

1. Image capture from the camera
2. Board state extraction via the vision module
3. Candidate move generation and evaluation by the chess engine
4. Instruction generation by the VLM model
5. Robotic execution via the VLA model
6. Post-execution validation

---

# 4. Evaluation Protocols

The system will be evaluated across several metrics.

### Move Accuracy

Percentage of robot moves that correctly match the intended instruction.

### Manipulation Success Rate

Percentage of attempts where the robot successfully grasps and places the correct piece.

### Instruction Robustness

Performance when commands are phrased in different styles, such as:

* formal instructions
* casual language
* simplified language

### Self-Correction Efficiency

Number of retries required to successfully complete an action.

### System Latency

End-to-end time from perception to completed move.

### Gameplay Balance

Average game length and move diversity when sampling from top-ranked Stockfish moves, measuring how the move selection strategy affects the competitiveness and variability of gameplay.

# 6. Why This Project Matters

Modern robotic manipulation models are increasingly capable of executing complex actions, but they often lack robust mechanisms for high-level planning and task reasoning.

This project explores how Vision-Language Models can serve as reasoning layers that guide robotic action models, enabling robots to perform structured tasks in a more autonomous and adaptive way.

While chess provides a controlled and structured test environment, the proposed architecture can generalise to many manipulation-heavy domains, such as:

* household tasks (for example folding laundry or organising objects)
* cooking tasks following natural-language recipes
* assembly and tool-use workflows
* structured industrial manipulation tasks

By demonstrating a modular pipeline connecting perception, reasoning, action, and validation, this project provides a reusable framework for building more capable robotic systems.
